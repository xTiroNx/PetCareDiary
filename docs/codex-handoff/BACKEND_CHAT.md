# Backend Chat Handoff

Use this file to start a Codex chat focused only on backend work.

## Scope

Work in `apps/backend` and `prisma`. Avoid frontend changes unless API response shape changes require type/UI updates and the user approves.

Backend responsibilities:
- Telegram initData auth.
- Access control and paid access checks.
- Pets and diary CRUD.
- Reminders storage and Telegram bot delivery.
- Reports and PDF export.
- Telegram Stars payments.
- Admin access management.

## Key Files

- `apps/backend/src/app.ts`
- `apps/backend/src/config/env.ts`
- `apps/backend/src/middlewares/`
- `apps/backend/src/routes/`
- `apps/backend/src/services/`
- `apps/backend/src/utils/`
- `prisma/schema.prisma`
- `prisma/migrations/`

## Security Rules

- Backend must validate Telegram initData using bot token.
- Backend must never trust `userId` from frontend.
- All protected endpoints must load current user from verified initData.
- Paid access must be granted only after Telegram webhook `successful_payment`.
- Admin access comes from configured Telegram numeric IDs.
- Webhook must validate `X-Telegram-Bot-Api-Secret-Token` when `TELEGRAM_WEBHOOK_SECRET` is set.

## Current Backend Features

- Express API with Zod validation.
- Prisma PostgreSQL models for users, pets, diary entries, reminders, payments, and report exports.
- Telegram Stars invoice creation.
- Payment webhook handling.
- Reminder scheduler sends Telegram bot messages.
- PDF report endpoint returns real PDF generated by backend.
- Rate limiting exists for sensitive route groups.

## Migrations

Use Prisma migrations only. Do not modify production database manually unless the user explicitly asks for an emergency fix.

Typical commands:

```sh
npm run prisma:generate
npm run prisma:migrate
npm run prisma:migrate:deploy
```

## Verification

Run:

```sh
npm --workspace apps/backend run typecheck
npm --workspace apps/backend run build
npm run security:smoke
```

Manual checks:
- `/health` returns `{"ok":true}`;
- expired users get `403 ACCESS_EXPIRED` for protected APIs;
- invalid initData is rejected;
- duplicate payment webhook does not grant access twice;
- PDF export contains real database records.
