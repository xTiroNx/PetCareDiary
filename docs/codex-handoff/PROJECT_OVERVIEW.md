# PetCare Diary - Project Overview

Use this file as the first context message for a new general Codex chat about the project.

## Product

PetCare Diary is a Telegram Mini App for cat/dog owners. It tracks feeding, symptoms, medicines, weight, reminders, reports, access status, and Telegram Stars payments.

MVP status:
- Telegram Mini App frontend is deployed on Netlify.
- Backend API is deployed on Render.
- PostgreSQL is managed by Render.
- Telegram auth uses verified WebApp initData.
- Reminders are delivered as Telegram bot messages.
- PDF reports are generated by backend with real database records.
- Multi-pet support exists with active pet selection and profile management.

## Production URLs

- Frontend: `https://petcare-diary.netlify.app`
- Backend: `https://petcare-diary-api-frankfurt.onrender.com`
- Backend health: `https://petcare-diary-api-frankfurt.onrender.com/health`

Do not put secrets in chat. Use placeholders such as `<BOT_TOKEN>`, `<DATABASE_URL>`, and `<RENDER_API_KEY>`.

## Stack

- Frontend: React, Vite, TypeScript, TailwindCSS, React Router, Zustand, TanStack Query.
- Telegram frontend bridge: raw `window.Telegram.WebApp` wrapper in `apps/frontend/src/utils/telegram.ts`.
- Backend: Node.js, TypeScript, Express, PostgreSQL, Prisma, Zod.
- Payments: Telegram Stars via Bot API invoice links and webhook confirmation.
- Hosting: Netlify frontend, Render backend/database.

## Safety Defaults

- Never trust `userId` from frontend; backend must derive user from validated Telegram initData.
- Do not grant paid access from frontend invoice callbacks.
- Grant paid access only after backend receives Telegram `successful_payment`.
- Keep real secrets only in Render/Netlify dashboards or local ignored `.env`.
- Rotate any token that was pasted into chat before production.
